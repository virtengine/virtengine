# VirtEngine Third-Party Security Audit RFP (Template)

## 1. Background
VirtEngine is a Cosmos SDK-based blockchain for decentralized cloud computing
with ML-powered identity verification (VEID). We are preparing for mainnet
launch and require third-party security audits across modules, cryptography,
ML models, infrastructure, and penetration testing.

## 2. Objectives
- Identify and validate security vulnerabilities in core components.
- Produce actionable remediation recommendations.
- Provide retest verification for fixed findings.
- Issue a final audit report and attestation letter.

## 3. Scope of Work
Provide proposals for one or more of the following scopes:
- Module security audit (Cosmos SDK modules).
- Cryptographic implementation audit.
- ML security audit (adversarial robustness and bias).
- Infrastructure security audit (Kubernetes/cloud/IAM).
- Penetration testing of externally exposed services.

Reference scope documents:
- _docs/audit/module-audit-scope.md
- _docs/audit/crypto-audit-scope.md
- _docs/audit/ml-audit-scope.md
- _docs/audit/infra-audit-scope.md
- _docs/audit/pentest-scope.md

## 4. Deliverables
- Detailed findings report with severity ratings.
- Reproduction steps and affected components.
- Remediation recommendations per finding.
- Retest verification results.
- Executive summary and attestation letter.

## 5. Timeline
Provide estimated start date, duration, and milestones for:
- Kickoff and access provisioning.
- Draft findings report.
- Final report and retest verification.

## 6. Required Qualifications
- Demonstrated experience with Cosmos SDK or similar chains.
- Proven cryptography audit capability.
- ML security assessment experience (if bidding on ML scope).
- References from prior blockchain or infrastructure audits.

## 7. Proposal Format
Please include:
- Firm overview and key personnel bios.
- Relevant audit experience and references.
- Proposed methodology and test approach.
- Timeline and staffing plan.
- Pricing structure (fixed + optional add-ons).
- Assumptions and out-of-scope items.

## 8. Confidentiality
All audit engagements require:
- NDA execution before access.
- Secure handling of sensitive materials.
- Agreement on coordinated disclosure.

## 9. Submission Details
- Submission deadline: [DATE]
- Contact: audit@virtengine.io
- Submission format: PDF + spreadsheet for pricing

## 10. Evaluation Criteria
Vendors will be evaluated using:
- Cosmos SDK experience
- Cryptography expertise
- ML security capability
- Timeline fit
- Cost
- Reputation and references

Refer to _docs/audit/vendor-evaluation.yaml for scoring weights.
